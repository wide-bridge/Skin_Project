# 05_작업지시서_(React_FastAPI_UI_API_구현범위정의)

## 1. 목적

피부질환 진단 서비스를 실제 사용 가능한 형태로 연결하기 위해 React 프론트엔드와 FastAPI 백엔드의 구현 범위를 정의한다.

## 2. UI 방향

`스킨케어.hwp`를 참고 자료로 삼아 아래 기능을 구현 범위에 포함한다.

- 피부질환 이미지 업로드
- 진단 결과 확인
- 챗봇형 상담 인터페이스
- 피부상담 대화 흐름
- HITL 개입 가능 상태 표시

단, 아래는 후속 논의로 남긴다.

- 상세 UX 카피
- 화면 세부 레이아웃
- 상담 흐름의 미세 인터랙션
- 운영자 리뷰 화면의 세부 동선

## 3. React 범위

- 앱 라우팅
- 업로드 페이지
- 결과 페이지
- 챗봇 UI
- 상담 이력 표시
- HITL 상태 배지 또는 알림

## 4. FastAPI 범위

- `/diagnosis/infer`
- `/chat/derma`
- `/chat/plastic`
- `/rag/explain`
- `/hitl/request`
- `/session/*`

엔드포인트 명칭은 후속 조정 가능하나, 기능 범위는 초기부터 포함한다.

## 5. API 응답 원칙

- 구조화 JSON 우선
- 프론트엔드가 렌더링 가능한 일관된 필드 유지
- 진단 결과와 상담 결과를 구분된 payload로 전달
- HITL 필요 여부를 명시 필드로 제공

## 6. 완료 기준

- UI가 구현 범위에 포함됨을 명시
- React/FastAPI 역할 구분 완료
- 향후 UX 세부화 여지를 남긴 범위 정의 완료
