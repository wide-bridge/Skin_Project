# 01_작업지시서_(프로젝트정의_및_범위확정)

## 1. 목적

이번 프로젝트는 기존 피부분석 중심 VLM 논의를 피부질환 진단 중심으로 전환하여, `피부질환 진단 + 챗봇 + 피부상담 + HITL`이 결합된 서비스형 AI 시스템으로 정의한다.

이번 01 단계의 목표는 아래와 같다.

- 프로젝트 목적과 범위를 문서 기준으로 고정
- VLM, RAG, Vision, HITL, UI/API의 역할 구분
- 구현 우선순위와 후속 작업 방향 확정
- 02 이후 단계가 바로 수행될 수 있도록 입력/출력 기준 정리

## 3. 프로젝트 정의

이 프로젝트는 단일 모델 실험이 아니라 아래를 포함하는 서비스 프로젝트다.

- 피부질환 진단 VLM
- 일반 피부상담 챗봇
- 진단 결과 설명 RAG
- 성형/미용 상담 확장 가능 구조
- HITL 검토 흐름
- React UI와 FastAPI API 서버

## 4. 현재 범위

### In Scope

- 피부질환 진단 중심 VLM 프로젝트
- 피부질환 이미지 데이터 구조 정리
- 질환 ontology 정리

- RAG 확장 전제 설계

- HITL 포함 운영 구조 반영

## 6. 핵심 차이점

기존 논의 대비 이번 프로젝트의 차이:

- 주제가 `피부분석`에서 `피부질환 진단`으로 이동
- 단일 VLM 실험이 아니라 서비스 전체를 염두에 둔 설계
- VLM이 본체이되, RAG와 비전 분기를 후속 확장 축으로 명시
- UI와 HITL을 구현 범위에 포함

## 7. 01 단계에서 실제로 수행할 작업

이번 단계에서 바로 수행해야 하는 작업은 아래와 같다.

1. 프로젝트 한 줄 정의 확정
- 이 프로젝트를 `피부질환 진단 VLM 기반 상담 서비스`로 명시

2. 기능 축 구분
- `Core`: VLM
- `Explain`: Diagnosis RAG
- `Consult`: General Derma / Plastic RAG
- `Fallback/Compare`: Vision branch
- `Service`: React + FastAPI
- `Safety/Ops`: HITL

3. 구현 우선순위 확정
- 1차: VLM
- 2차: 진단 설명 RAG
- 3차: 일반 상담 RAG
- 4차: 성형 상담 RAG
- 5차: Vision branch
- 6차: HITL 고도화

4. 문서 후속 분해 기준 확정
- 02: 시스템 아키텍처
- 03: 데이터 및 VLM 학습 기반
- 04: RAG / Vision / HITL 확장 구조
- 05: React / FastAPI 구현 범위
