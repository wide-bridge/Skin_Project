# 01_작업지시서_(프로젝트정의_및_범위확정)

## 1. 목적

이번 프로젝트는 기존 피부분석 중심 VLM 논의를 피부질환 진단 중심으로 전환하여, `피부질환 진단 + 상담 + 설명 + HITL 확장 가능 구조`를 갖는 서비스형 AI 시스템으로 정의한다.

이번 01 단계의 목표는 아래와 같다.

- 프로젝트 목적과 범위를 문서 기준으로 고정
- VLM, baseline vision, RAG, HITL, UI/API의 역할 구분
- 구현 우선순위와 후속 작업 방향 확정
- 02 이후 단계가 바로 수행될 수 있도록 입력/출력 기준 정리

## 2. 선행 조건

- 외부 환경키 위치 확인 완료

## 3. 프로젝트 정의

이 프로젝트는 단일 모델 실험이 아니라 아래를 포함하는 서비스 프로젝트다.

- 피부질환 진단 VLM
- baseline vision classifier 비교 축
- 진단 결과 설명 RAG
- 일반 피부상담/성형상담 확장 가능 구조
- HITL 검토 흐름
- React UI와 FastAPI API 서버

이번 단계에서 확정한 프로젝트 한 줄 정의:

`피부질환 진단 VLM 기반 상담 서비스`

## 4. 현재 범위

### In Scope

- 피부질환 진단 중심 VLM 프로젝트
- 피부질환 이미지 데이터 구조 정리
- 질환 ontology 정리
- 진단 결과의 구조화 출력 설계
- baseline vision 비교 경로 반영
- RAG 확장 전제 설계
- React 프론트엔드와 FastAPI 백엔드 구조 반영
- HITL 포함 운영 구조 반영

### Out of Scope for Now

- 상세 화면 UX 확정
- 세부 상담 시나리오 확정
- 의료 운영정책 확정
- 실서비스 배포 인프라 확정
- 최종 의료 책임 체계 확정

## 5. 구현 순서 원칙

이번 프로젝트는 아래 순서로 진행한다.

- `04`: 데이터/학습셋 준비 + baseline vision + Qwen3-VL LoRA/QLoRA 모델 실험
- `05`: RAG 데이터정제 및 검색기반구축
- `06`: 최소 API/챗봇 연결 준비
- `07`: RAG / Vision / HITL 확장구조정의
- `08`: React / FastAPI / UI / API 구현범위정의

즉, 1차 구현은 `04~06`에서 마무리하고, `07~08`은 후속 확장 단계로 둔다.

## 6. 이번 단계에서 확정한 기능 축

- `Core`: Qwen3-VL 계열 모델
- `Compare`: baseline vision classifier
- `Explain`: Diagnosis RAG
- `Consult`: General Derma / Plastic RAG
- `Service`: React + FastAPI
- `Safety/Ops`: HITL
