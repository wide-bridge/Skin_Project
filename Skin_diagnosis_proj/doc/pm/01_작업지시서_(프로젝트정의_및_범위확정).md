# 01_작업지시서_(프로젝트정의_및_범위확정)

## 1. 목적

이번 프로젝트는 기존 피부분석 중심 VLM 논의를 피부질환 진단 중심으로 전환하여, `피부질환 진단 + 챗봇 + 피부상담 + HITL`이 결합된 서비스형 AI 시스템으로 정의한다.

핵심 목표:

- 피부질환 이미지를 이해하는 VLM 기반 진단 기능 구축
- 진단 결과를 설명하는 챗봇형 응답 구조 설계
- 피부질환 상담과 성형/미용 상담이 확장 가능한 구조 확보
- 향후 React + FastAPI 기반 제품 UI/서비스 구현 가능 상태 확보

## 2. 프로젝트 정의

이 프로젝트는 단일 모델 실험이 아니라 아래를 포함하는 서비스 프로젝트다.

- 피부질환 진단 VLM
- 일반 피부상담 챗봇
- 진단 결과 설명 RAG
- 성형/미용 상담 확장 가능 구조
- HITL 검토 흐름
- 웹 UI와 API 서버

## 3. 현재 범위

### In Scope

- 피부질환 진단 중심 VLM 프로젝트
- 피부질환 이미지 데이터 구조 정리
- 질환 ontology 정리
- 진단 결과의 구조화 출력 설계
- RAG 확장 전제 설계
- 비전 전용 분기 추가 가능성 반영
- React 프론트엔드와 FastAPI 백엔드 구조 반영
- HITL 포함 운영 구조 반영

### Out of Scope for Now

- 세부 화면 UX 확정
- 세부 상담 시나리오 확정
- 의료 운영정책 확정
- 실서비스 배포 인프라 확정
- 최종 의료적 책임 체계 확정

## 4. 참고 자산

이번 프로젝트는 아래 자료를 참고하되, 독립 프로젝트로 진행한다.

- `Skin_Proj/doc/local_only`
- `Skin_Proj/doc/pm`
- `스킨케어.hwp`
- 과거 `Derma_AI` 및 관련 피부질환/Q&A 데이터 구조

## 5. 핵심 차이점

기존 논의 대비 이번 프로젝트의 차이:

- 주제가 `피부분석`에서 `피부질환 진단`으로 이동
- 단일 VLM 실험이 아니라 서비스 전체를 염두에 둔 설계
- VLM이 본체이되, RAG와 비전 분기를 후속 확장 축으로 명시
- UI와 HITL을 구현 범위에 포함

## 6. 성공 기준

최소 성공 기준:

- 프로젝트 범위 문서화 완료
- VLM, RAG, Vision, HITL, UI/API 역할 구분 완료
- 단계별 작업지시서 분해 완료
- 후속 구현에 필요한 구조와 우선순위 합의 완료
