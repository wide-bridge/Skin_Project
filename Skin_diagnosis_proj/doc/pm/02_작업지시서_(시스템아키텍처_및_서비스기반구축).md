# 02_작업지시서_(시스템아키텍처_및_서비스기반구축)

## 1. 목적

피부질환 진단 서비스를 위한 기본 시스템 아키텍처를 정의하고, 이후 구현이 가능한 폴더/모듈 단위를 확정한다.

## 2. 전체 아키텍처

```text
React Web UI
  -> FastAPI Gateway
    -> Orchestrator / Router
      -> VLM Service
      -> Vision Service
      -> RAG Service
      -> HITL Service
    -> Logging / Session / Metadata
```

## 3. 컴포넌트 정의

### 3.1 React Web

- 이미지 업로드
- 진단 결과 카드 표시
- 챗봇형 질의응답 인터페이스
- 상담 이력 보기
- HITL 검토 요청 상태 표시

### 3.2 FastAPI

- 이미지 추론 API
- 상담 API
- RAG 검색/응답 API
- HITL 요청 API
- 세션 및 로그 API

### 3.3 Orchestrator

- 입력 유형 판별
- 이미지/텍스트/혼합형 요청 라우팅
- 진단 후 설명 호출
- 상담형 질의 분기
- 실패 시 fallback 경로 제어

### 3.4 VLM Service

- 피부질환 이미지 기반 구조화 추론
- 질환 후보, 신뢰도, 근거 필드 생성
- JSON strict parsing 가능한 출력 제공

### 3.5 Vision Service

- 필요 시 순수 비전 모델 분기 운영
- VLM 결과 검증 또는 baseline 비교
- 성능/속도 비교용 실험 경로 제공

### 3.6 RAG Service

- 진단 설명용 RAG
- 일반 피부질환 상담 RAG
- 성형/미용 상담 RAG

### 3.7 HITL Service

- 사람이 확인해야 하는 케이스 표기
- 리뷰 큐 생성
- 수정 의견 기록
- 추후 학습 데이터 피드백 연결 가능 구조

## 4. 구현 원칙

- 서비스 코어는 FastAPI 기준으로 분리
- 웹 UI는 React 기준으로 별도 앱 구성
- AI 기능은 서비스 단위 모듈로 분리
- RAG와 VLM은 서로 결합하되 독립 테스트 가능해야 함
- HITL은 나중에 UX를 구체화하더라도 구조는 초기에 반영

## 5. 완료 기준

- 시스템 컴포넌트 역할 정의 완료
- 모듈 분리 기준 정의 완료
- React/FastAPI 포함 서비스 구조 문서화 완료
- 오케스트레이션을 붙일 수 있는 뼈대 정의 완료
