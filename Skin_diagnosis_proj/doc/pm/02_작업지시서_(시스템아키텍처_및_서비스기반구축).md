# 02_작업지시서_(시스템아키텍처_및_서비스기반구축)

## 1. 목적

피부질환 진단 서비스를 위한 기본 시스템 아키텍처를 확정하고, 이후 구현이 가능한 앱/서비스/데이터/라우팅 경계를 문서 기준으로 고정한다.

이번 02 단계의 목표는 아래와 같다.

- React + FastAPI 기반 서비스 구조 확정
- VLM, baseline vision, RAG, HITL의 모듈 경계 확정
- 오케스트레이션 흐름 확정
- 04~06 1차 구현이 가능한 구조 고정

## 2. 선행 조건

01 단계는 완료 상태를 전제로 한다.

- 프로젝트 한 줄 정의 확정 완료
- In Scope / Out of Scope 확정 완료
- 기능 축 구분 완료
- 구현 우선순위 확정 완료

## 3. 전체 아키텍처

```text
React Web UI
  -> FastAPI Gateway
    -> Orchestrator / Router
      -> VLM Service
      -> Vision Service
      -> RAG Service
      -> HITL Service
    -> Logging / Session / Metadata
```

## 4. 이번 단계에서 확정한 아키텍처 적용 순서

1차 구현 적용 순서:

- `04`: 모델 중심 구현
  - baseline vision classifier
  - Qwen3-VL LoRA/QLoRA
- `05`: RAG Service 기반 추가
- `06`: 최소 API/챗봇과 연결

후속 확장:

- `07`: Vision branch 정교화, HITL, 고급 오케스트레이션 확장
- `08`: React/FastAPI 서비스 범위 고도화

## 5. 컴포넌트 정의 요약

- `React Web`: 업로드, 결과, 챗봇 UI
- `FastAPI Gateway`: 추론/상담/HITL/session API
- `Orchestrator`: 입력 판별과 라우팅
- `VLM Service`: Qwen3-VL 계열 본체 진단
- `Vision Service`: baseline/fallback/compare
- `RAG Service`: 설명/상담 검색
- `HITL Service`: 사람 검토 경로

## 6. 아키텍처 결정 사항

- VLM은 본체 모듈
- baseline vision은 비교 및 보조 판단 축
- RAG는 VLM 뒤 설명/상담 레이어
- 06 단계까지는 full chatbot보다 최소 API/연결 루프 우선
- Vision / HITL / 고도화 UI는 후속 단계로 이월 가능

## 7. 완료 기준

아래를 만족하므로 02 단계는 완료로 처리한다.

- 시스템 컴포넌트 역할 정의 완료
- 모듈 분리 기준 정의 완료
- 04~06 1차 구현 라인이 아키텍처에 반영됨
- 07~08 후속 확장 라인이 분리됨
- 03 이후 단계에서 바로 구현이 가능함
