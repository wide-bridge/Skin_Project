## 2. 현재 판단
- 이번 종료는 단순 Python 예외가 아니라 OS 레벨 크래시다.
- 원인 후보는 다음 순서로 본다.
  1. `Qwen3-VL-8B + device_map=auto` 조합에서 GPU/CPU 메모리 오프로딩이 과하게 걸리며 시스템 메모리 관리가 불안정해진 경우

## 3. 재발 방지용 안전 실행안
### 3.1 즉시 적용
- 8B one-batch 학습 반복 중단
- `device_map=auto` 기반 대형 모델 smoke test 중단
- Hugging Face 네트워크 접근 없이 캐시만 사용하는 `local_files_only=true` 유지

### 3.2 실행 순서
1. `2B smoke-safe`로 로드/추론/forward 경로만 먼저 확인
2. `2B smoke-safe`에서 forward, loss, backward, optimizer, checkpoint save/load 확인
3. `2B smoke-safe`에서 Scenario 01/02/03 각각 1건 추론 확인
4. 위가 모두 안정적일 때만 8B 추론 검증 진행
5. 8B는 추론부터 올리고, 학습은 마지막 단계에서 다시 시도

### 3.3 실행 규칙
- smoke test는 작은 모델, 작은 입력, 단일 GPU로만 실행
- page file 자동 관리 또는 충분한 수동 크기 유지
- 백그라운드 GPU 사용 앱 종료 후 실행
- 장시간 학습 전 GPU 드라이버 오류 로그와 시스템 이벤트 로그를 먼저 확인
