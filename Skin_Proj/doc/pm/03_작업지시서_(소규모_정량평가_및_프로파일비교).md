# 03_작업지시서_(소규모_정량평가_및_프로파일비교)

## 1. 목적

03 단계의 목적은 `Qwen3-VL + LoRA` 파이프라인의 연결 여부를 넘어,  
작은 규모에서 반복 가능한 정량 평가 기준을 세우는 것이다.

이번 단계에서는 아래를 확인한다.

- `primary_4b` 기준 배치 추론 안정성
- `quantized_4b` 비교 프로파일의 실행 가능성
- 시나리오 `01/02/03`별 JSON 출력 안정성
- key exact match / label range validation / exact match 집계
- 이후 04 단계 성능 개선으로 넘어갈 수 있는 최소 평가 기준 수립

## 2. 전제 상태

03은 아래가 완료된 상태에서 시작한다.

- `00` GitHub 및 형상관리 기반 완료
- `01` 데이터/구조/validator/config 기반 완료
- `02` `primary_4b` one-batch smoke test 및 `01/02/03` 최소 추론 검증 완료

즉 03은 연결 검증 단계가 아니라 평가 단계이다.

## 3. 기본 모델 정책

이번 단계의 기본 정책은 아래와 같다.

- `primary_4b`: 기본 평가 프로파일
- `quantized_4b`: 비교 후보 프로파일
- `smoke_safe_2b`: 필요 시 복구/대조용 프로파일
- `deferred_8b`: 로컬 LoRA 학습 금지 프로파일

추가 원칙:

- `primary_4b` 평가가 03의 기준 경로이다.
- `quantized_4b`는 비교 후보이며, 실패해도 03 전체 실패로 보지 않는다.
- `quantized_4b` 실패 시 실패 원인과 로그를 남기고, `primary_4b` 평가를 기준으로 계속 진행한다.

## 4. 작업 범위

### 4.1 배치 평가 스크립트 추가

단일 이미지 CLI 추론이 아니라, 시나리오별 샘플 목록을 받아 반복 추론하는 평가 스크립트를 추가한다.

권장 경로:

```text
inference/batch_eval.py
```

입력:

- profile
- scenario list (`01`, `02`, `03`)
- 평가 샘플 목록 또는 샘플 수

출력:

- JSON summary
- Markdown summary

### 4.2 평가 샘플 정책

각 시나리오에서 최소 `10~30장` 수준의 샘플을 사용한다.

기본 원칙:

- 동일 `person_id` 편향 최소화
- `01/02/03` 모두 포함
- 정답 라벨이 존재하는 샘플만 사용

### 4.3 검증 항목

각 추론 결과에 대해 아래를 집계한다.

- JSON parsing success rate
- key exact match rate
- label range valid rate
- exact match rate
- scenario별 결과 분리
- profile별 결과 분리

필수 key:

- `forehead_wrinkle`
- `forehead_pigmentation`

### 4.4 프로파일 비교

최소 비교 대상:

- `primary_4b`
- `quantized_4b`

비교 항목:

- model load 가능 여부
- 1장당 평균 추론 시간
- JSON parsing success rate
- key/range validity
- exact match rate

### 4.5 결과 로그 분리

03 평가 결과는 smoke test 로그와 분리한다.

권장 로컬 산출물 경로:

```text
outputs/eval/phase03_eval_summary.json
outputs/eval/phase03_eval_summary.md
```

정책:

- `outputs/eval/phase03_eval_summary.*`는 로컬 산출물이며 GitHub 커밋 금지
- GitHub에는 평가 요약만 아래 문서에 반영
  - `docs/dev/current_status.md`
  - `docs/dev/next_actions.md`

구분 원칙:

- `validation_report`: 데이터 검증 전용
- `smoke_test_log`: 연결 검증 전용
- `phase03_eval_summary`: 소규모 정량평가 전용

## 5. 구현 원칙

- 공용 config 기준 유지
- `config.local.yaml`은 Git 추적 금지 유지
- Prompt 본문 하드코딩 금지
- 배치 평가 중 생성되는 임시 결과물은 Git 추적 제외
- GitHub에는 요약 문서만 반영

## 6. 제외 범위

이번 단계에서 하지 않는 것:

- ROI crop
- prompt tuning
- data augmentation
- 장기 epoch 학습
- 8B local LoRA 학습
- UI / 챗봇 제품화
- RAG / Vector DB
- 추천 시스템

## 7. 완료 기준

아래를 만족하면 03 완료 후보로 본다.

- `primary_4b` 배치 평가 성공

- `docs/dev/next_actions.md` 반영 완료
