# 05_작업지시서_(정확도개선_및_능동학습_적용판단)

## 1. 현재 상태

이 문서는 `Skin_Proj`의 Qwen3-VL 기반 피부 분석 실험을 여기서 일시 중단하고, 나중에 다시 이어가기 위한 기준 문서이다.

현재까지의 핵심 결론은 다음과 같다.

- `00` GitHub 운영 기준 수립: 완료
- `01` Vision Agent 기반 구축: 완료
- `02` Qwen3-VL + LoRA smoke path: 완료
- `03` 소규모 정량평가: 완료
- `04` all-zero collapse 원인 파악 및 1차 제거: 완료
- `05` 정확도 개선 및 능동학습 적용 판단: 보류

`04`에서 확인된 구조적 문제는 다음 두 가지였다.

- inference prompt가 0 값 JSON 예시를 포함하여 zero prediction bias를 만들 수 있었다.
- training path에서 assistant JSON token에만 loss가 걸리지 않고 prompt token까지 loss 대상이 되는 문제가 있었다.

수정 후 all-zero collapse와 single-value collapse는 30샘플 기준으로 더 이상 관찰되지 않았다. 따라서 현재 문제는 collapse가 아니라 낮은 정확도 문제로 전환되었다.

## 2. 05의 원래 목적

05의 목적은 바로 장시간 LoRA 학습을 돌리는 것이 아니라, 정확도 개선을 위한 데이터 전략과 능동학습 가능성을 판단하는 것이었다.

핵심 목표는 다음과 같았다.

- 신뢰성 있는 데이터를 정의하고 확보한다.
- trusted / uncertain / hard / invalid sample을 분류한다.
- 어떤 샘플을 다음 학습에 넣어야 정확도 개선 가능성이 있는지 판단한다.
- GPU 비용을 고려하여 1 step -> 3 step -> 10 step -> 30 step 식으로 lazy하게 확장한다.
- 장시간 학습은 데이터/샘플 전략이 선명해진 뒤에만 수행한다.

## 3. 신뢰성 있는 데이터의 정의

이 프로젝트에서 신뢰성 있는 데이터는 단순히 모델이 맞힌 데이터가 아니다.

신뢰성 있는 데이터는 다음 조건을 만족해야 한다.

- 라벨링 기준이 일관적이어야 한다.
- 같은 피부 상태에 대해 같은 기준으로 class 0~4가 부여되어야 한다.
- 사람마다 또는 시나리오마다 라벨 기준이 크게 흔들리지 않아야 한다.
- label과 image가 정상적으로 매칭되어야 한다.
- class별 분포가 학습 가능한 수준으로 확보되어야 한다.
- 틀린 라벨이 있더라도 체계적으로 틀린 것인지, 무작위로 흔들린 것인지 구분 가능해야 한다.

사용자 판단:

- 데이터는 성형외과 의사가 눈으로 판정한 것으로 보인다.
- 같은 사람이 아닐 수 있어 체계적 오류와 비체계적 오류가 모두 가능하다.
- 일부 라벨은 동일 라벨을 복사한 구조일 수 있다.
- 따라서 모델 성능 이전에 라벨 기준의 일관성을 먼저 보아야 한다.

## 4. 주름 데이터에 대한 중간 판단

검토 대상 주름 target은 다음과 같다.

- `forehead_wrinkle`: 이마 주름
- `glabellus_wrinkle`: 미간 주름
- `l_perocular_wrinkle`: 좌측 눈가 주름
- `r_perocular_wrinkle`: 우측 눈가 주름

중간 판단은 다음과 같다.

- 이마, 미간, 좌우 눈가 주름은 서로 완전히 같은 target은 아니지만 상관성이 높다.
- 좌우 눈가 주름은 매우 유사하여 하나의 `perocular_wrinkle`로 통합하는 전략도 가능하다.
- 4 target을 모두 분리하면 세부 분석은 가능하지만 학습 난이도와 평가 복잡도가 증가한다.
- 3 target으로 줄이면 `forehead_wrinkle`, `glabellus_wrinkle`, `perocular_wrinkle` 구조가 된다.
- 다만 현재 핵심 문제는 target 수가 아니라 좋은 데이터와 학습 전략이다.

## 5. 과거 성공 실험과 현재 실패 원인 후보

사용자 회고 기준으로 과거 성공 조건은 다음과 같다.

- ROI bbox crop을 사용했다.
- 단순 freeze만 한 실험은 반복적으로 성능이 좋지 않았다.
- 성공 실험은 일부 동결 해제 또는 더 깊은 fine-tuning이 있었을 가능성이 높다.
- augmentation을 적극 사용했다.
- 가장 적은 class는 약 10배, 가장 많은 class는 약 1.5배 수준으로 증강했다.
- 1 epoch에 약 50분 정도 걸렸고, 총 15시간 정도 학습했다.
- 주름 분류는 약 90% 수준까지 나왔던 것으로 기억된다.
- 잡티/색소 계열 회귀 또는 세부 예측은 당시에도 좋지 않았다.

현재 Qwen3-VL LoRA 실험에서 성능이 낮은 이유 후보는 다음과 같다.

- 학습량이 너무 작다.
- freeze/unfreeze 전략이 과거 성공 실험과 다르다.
- augmentation이 충분하지 않다.
- ROI 전략이 현재 데이터/모델 조합에서 제대로 먹히지 않았다.
- VLM이 전체 얼굴 이미지만으로 세부 피부 등급을 안정적으로 구분하기 어렵다.
- 라벨 기준 자체가 모델이 학습하기 어려운 방식일 수 있다.

## 6. Lazy Active Learning 전략

05에서 제안했던 전략은 다음과 같다.

1. 먼저 평가 샘플을 분류한다.
2. `trusted`, `uncertain`, `hard`, `invalid` queue를 만든다.
3. class / scenario / target별로 신뢰성 있는 데이터가 얼마나 있는지 확인한다.
4. 부족한 class와 scenario를 우선 보강한다.
5. 필요한 경우에만 1 step 학습을 수행한다.
6. 다시 소규모 평가를 수행한다.
7. 불확실성이 줄어들면 3 step으로 확장한다.
8. GPU 시간과 VRAM 부담을 고려해 추가 확장을 제한한다.

신뢰성 있는 데이터 확보 목표는 다음 기준으로 논의되었다.

- scenario: 3개
- class: 5개
- 각 scenario x class 조합마다 최소 3개 확보
- target 1개 기준 최소 목표: `3 scenarios x 5 classes x 3 = 45`
- target이 2개면 최소 90개 수준
- target이 4~5개면 중복 이미지 허용 여부를 별도로 기록해야 한다.

## 7. GPU-aware Lazy Training 기준

장시간 학습은 자동으로 시작하지 않는다.

학습 전에는 반드시 다음을 예측하고 보고해야 한다.

- profile
- batch size
- max steps
- step당 평균 시간
- 예상 학습 시간
- 예상 checkpoint 저장 시간
- 예상 평가 시간
- 총 예상 시간 범위
- 권장 timeout

timeout 기준:

- 예상 총 시간 x 1.5 이상
- 최소 30분
- 4B small-run은 60분 이상 허용
- 긴 실험은 학습 timeout과 평가 timeout을 분리
- timeout 값은 코드에 하드코딩하지 않고 config 또는 실행 인자로 관리

## 8. 05 보류 사유

05는 여기서 완료하지 않고 보류한다.

보류 이유는 다음과 같다.

- Qwen3-VL 기반 VLM 실험은 구조 검증과 collapse 제거까지는 완료했다.
- 그러나 정확도 개선을 위해서는 단순 LoRA step 확대보다 데이터와 학습 전략 재기획이 필요하다.
- 사용자가 새 방향으로 피부진단 데이터와 피부과 상담 데이터를 활용한 프로젝트를 별도로 진행하기로 결정했다.
- 피부진단 데이터는 합성 데이터 성격이 있어 VLM 구조 검증에는 더 적합할 가능성이 있다.
- 피부과 상담 데이터는 이미지 분류와 분리하여 RAG/상담 응답 기획으로 다루는 것이 더 적절하다.

따라서 `Skin_Proj`는 여기서 멈추고, 새 프로젝트 `Skin_diagnosis_proj`에서 피부진단/상담 데이터 기반으로 다시 기획한다.

## 9. 나중에 재개할 때 확인할 것

`Skin_Proj`를 다시 열 경우 다음 순서로 재개한다.

1. `docs/dev/current_status.md`와 이 문서를 먼저 읽는다.
2. `04`에서 수정된 prompt bias 제거와 assistant-only loss mask가 유지되는지 확인한다.
3. `outputs/eval/*`, `outputs/debug/*`, `lora/*`, `config.local.yaml`, `.env`는 GitHub 커밋 대상이 아님을 재확인한다.
4. 신뢰성 있는 데이터 정의를 모델 정답 여부가 아니라 라벨 일관성 기준으로 다시 적용한다.
5. 과거 성공 실험의 핵심 조건인 ROI, augmentation, unfreeze, 장시간 학습을 별도 실험안으로 정리한다.
6. VLM을 계속할 경우 synthetic/consultation 기반 새 데이터와 기존 피부 등급 데이터를 섞지 말고 목적별로 분리한다.
